# 功能 (FeaturesPage) 设计规范

## 布局结构

- 顶部：可折叠大标题栏，标题 "功能"（`TopAppBar`（顶部大标题栏） + `MiuixScrollBehavior`（标题栏跟随滚动自动折叠））
- 内容：垂直滚动列表（`LazyColumn`（按需加载的滚动列表）），水平内边距 12dp（dp 是与屏幕密度无关的尺寸单位）
- 八个区域（按顺序）：隐私、背屏天气卡片（含刷新间隔和优先级子设置）、歌词显示、壁纸相关（导航入口）、背屏常亮、背屏 DPI、投屏设置
- 壁纸相关功能已拆分到独立子页面 `WallpaperPage`，通过 `SuperArrow`（带箭头的可点击项） 导航进入

**源文件**: `app/src/main/kotlin/org/pysh/janus/ui/FeaturesPage.kt`

## 组件

| 组件 | 用途 |
|------|------|
| `TopAppBar` | 可折叠大标题栏 |
| `Card` x 8 | 八个功能区域的容器 |
| `SuperSwitch` x 4 | 隐私开关、天气卡片开关、背屏常亮开关、投屏保活开关 |
| `SuperArrow` x 8 | 天气刷新间隔（带 Slider）、天气优先级（带 Slider）、歌词淡入时长（带 Slider）、歌词检测阈值（带 Slider）、壁纸相关（导航）、唤醒间隔（带 Slider）、背屏 DPI（带 Slider）、投屏旋转 |
| `Slider` x 6 | 天气刷新间隔滑块、天气优先级滑块、歌词淡入时长滑块、歌词检测阈值滑块、唤醒间隔滑块、DPI 滑块 |
| `TextButton` x 4 | 重置默认间隔、设置 DPI、重置 DPI、重置旋转 |
| `SuperDialog` x 3 | 唤醒间隔精确输入弹窗、DPI 精确输入弹窗、投屏旋转选择弹窗 |
| `SuperRadioButton` x 3 | 旋转弹窗中的三个选项（不旋转、向左、向右） |
| `TextField` | 输入弹窗内的数字输入框 |

## 视觉规范

### 区域布局

- 每个区域直接使用 Card 包裹
- 第一个 Card：上边距 12dp + 下边距 12dp
- 后续 Card：仅下边距 12dp

### 区域一：隐私

- 单个 `SuperSwitch`（开关控件）
  - 标题："禁止使用统计上报"
  - 摘要：根据开关状态显示不同文本
  - 开关操作后弹出 Toast（屏幕底部短暂弹出的提示消息） 提示

### 区域二：背屏天气卡片

- `SuperSwitch`（开关控件）
  - 标题："背屏天气卡片"
  - 摘要：根据开关状态显示不同文本（"服务助手中显示天气卡片" / "不显示天气卡片"）
  - 开关操作后通过 root 创建/删除标志文件（创建时附带 `chcon` 修正 SELinux 上下文），自动重启背屏生效，弹出 Toast 提示
  - 需要 Xposed Hook 生效：在服务助手面板注入独立天气 MAML 卡片，通过 ContentProviderBinder 从 MIUI 天气获取数据
- 以下子设置仅在天气卡片开启时显示：
- `SuperArrow` + `Slider` 组合：刷新间隔
  - 标题："刷新间隔"
  - 摘要：显示当前分钟值（如"30 分钟"）
  - Slider 范围：10 ~ 120
  - 拖动结束后通过 `WhitelistManager.setWeatherRefreshInterval()` 持久化，并写入 flag 文件供 Hook 侧读取
- `SuperArrow` + `Slider` 组合：卡片优先级
  - 标题："卡片优先级"
  - 摘要：显示当前优先级值
  - Slider 范围：1 ~ 999
  - 拖动结束后通过 `WhitelistManager.setWeatherCardPriority()` 持久化

### 区域三：歌词显示

- `SuperArrow` + `Slider` 组合：淡入时长
  - 标题："淡入时长"
  - 摘要：显示当前毫秒值（如"700 ms"）
  - Slider 范围：100 ~ 2000
  - 拖动结束后通过 `WhitelistManager.setLyricFadeDuration()` 持久化，并写入 flag 文件供 Hook 侧读取
- `SuperArrow` + `Slider` 组合：检测阈值
  - 标题："检测阈值"
  - 摘要：显示当前秒数值（如"15 s"）
  - Slider 范围：1 ~ 60（UI 用秒，内部存储为毫秒）
  - 拖动结束后通过 `WhitelistManager.setLyricModeThreshold()` 持久化

### 区域四：壁纸相关（导航入口）

- 单个 `SuperArrow`
  - 标题："壁纸相关"
  - 点击导航到壁纸子页面（`WallpaperPage`）

### 区域五：背屏常亮

- `SuperSwitch`：背屏常亮开关
  - 标题："背屏常亮"
  - 摘要：根据开关状态显示不同文本
- `SuperArrow` + `Slider`（滑动条） 组合（`bottomAction`（底部附加内容） 模式）
  - 标题："唤醒间隔"
  - 摘要：显示当前秒数值
  - 点击箭头 -> 弹出 `SuperDialog`（弹窗组件） 精确输入
  - Slider 范围：1 ~ 300
  - Slider 拖动结束后持久化，若常亮服务运行中则热更新间隔
- 底部按钮行（`Row`（横向布局容器））
  - `TextButton`（文字按钮）："重置默认"，恢复到 DEFAULT_INTERVAL
  - 水平内边距 12dp，底部内边距 12dp，等权重

### 区域六：背屏 DPI

- `SuperArrow` + `Slider` 组合（bottomAction 模式）
  - 标题："背屏 DPI"
  - 摘要：显示当前 DPI 值，加载中时显示 "加载中..."
  - 点击箭头 -> 弹出 SuperDialog 精确输入
  - Slider 范围：100 ~ 800
- 底部按钮行（Row）
  - `TextButton`："设置 DPI"，使用 `ButtonDefaults.textButtonColorsPrimary()`（主题色按钮样式） 主色调
  - `TextButton`："重置"，默认色调
  - 间距 8dp，水平内边距 12dp，底部内边距 12dp，等权重

### 区域七：投屏设置

- `SuperArrow`：投屏旋转
  - 标题："投屏旋转"
  - 摘要：显示当前旋转状态（不旋转 / 向左旋转 90° / 向右旋转 90°）
  - 点击弹出旋转选择弹窗（SuperDialog + `SuperRadioButton`（单选按钮））
- `SuperSwitch`：投屏保活
  - 标题："投屏保活"
  - 摘要：根据开关状态显示不同文本
- 按钮行：`TextButton`"重置"，默认色调
  - 水平内边距 12dp，底部内边距 12dp，等权重
  - 点击调用 `DisplayUtils.resetRearRotation()` 清除残留旋转状态

### 弹窗规范

#### 输入弹窗（唤醒间隔、DPI）

两个输入 `SuperDialog` 结构一致：
- 标题 + 说明文字
- `TextField`（文本输入框）：仅允许数字输入（`filter { it.isDigit() }`（只允许输入数字）），宽度填满，底部间距 16dp
- 按钮行：取消 + 确认，间距 8dp，等权重
  - 确认按钮使用 `textButtonColorsPrimary()` 主色调
- 唤醒间隔弹窗：输入值 `coerceIn(1, 300)`（超出范围时自动取边界值，如输入 0 变成 1、500 变成 300）
- DPI 弹窗：输入值 `coerceIn(100, 800)`（超出范围时自动取边界值），确认后仅更新 Slider 值（不立即应用）

#### 旋转选择弹窗

- 标题："投屏旋转"，无说明文字
- 三个选项，每个用 `Card`（卡片容器） 包裹一个 `SuperRadioButton`：
  - "不旋转"（值 0），默认选中
  - "向左旋转 90°"（值 1）
  - "向右旋转 90°"（值 3）
- 前两个 Card 底部间距 8dp，最后一个无底部间距
- 选择后立即保存到 `WhitelistManager.setCastRotation()` 并关闭弹窗

## 交互

- 点击壁纸相关 -> 导航到 WallpaperPage 子页面
- 开启常亮 -> 启动 `ScreenKeepAliveService`，传入当前间隔
- 关闭常亮 -> 停止 `ScreenKeepAliveService`
- 设置 DPI -> 通过 `DisplayUtils.setRearDpi()` 应用，成功后 `RootUtils.restartBackScreen()` 重启背屏
- 重置 DPI -> 通过 `DisplayUtils.resetRearDpi()` 恢复默认，成功后重启背屏
- DPI 操作在 IO 线程（后台线程，不会卡住界面）执行，完成后更新 UI 状态并显示 Toast
- 投屏旋转点击弹出选择弹窗，选择后立即持久化到 `WhitelistManager`
- 投屏保活开关切换，立即持久化并显示 Toast
- 重置旋转 -> 通过 `DisplayUtils.resetRearRotation()` 清除残留旋转状态，在 IO 线程执行，Toast 反馈成功/失败

## 壁纸子页面 (WallpaperPage) 设计规范

### 布局结构

- 顶部：`SmallTopAppBar`（固定小标题栏），标题"壁纸相关"，带返回按钮（`MiuixIcons.Back`）
- 内容：`Column`（纵向布局容器） + 垂直可滚动（`verticalScroll`），水平内边距 12dp
- 两个 Card 区域：壁纸防打断 + 壁纸锁定、AI 壁纸自定义

**源文件**: `app/src/main/kotlin/org/pysh/janus/ui/WallpaperPage.kt`

### 组件

| 组件 | 用途 |
|------|------|
| `SmallTopAppBar` | 顶部标题栏 + 返回按钮 |
| `Card` x 2 | 两个功能区域的容器 |
| `SuperSwitch` x 3 | 壁纸防打断开关、壁纸锁定开关、视频循环开关 |
| `SuperArrow` x 1 | 自定义壁纸视频 |
| `TextButton` x 1 | 恢复原始壁纸 |

### 区域一：动态壁纸防打断 + 壁纸锁定

- `SuperSwitch`：动态壁纸防打断
  - 标题："动态壁纸防打断"
  - 摘要：根据开关状态显示不同文本（"遮盖时动态壁纸继续播放" / "遮盖时动态壁纸暂停"）
  - 开关操作后弹出 Toast 提示
  - 需要 Xposed Hook 生效：Hook `SubScreenLauncher.onPause()` 阻止背屏 MAML 壁纸引擎被销毁
- `SuperSwitch`：锁定壁纸切换
  - 标题："锁定壁纸切换"
  - 摘要：根据开关状态显示不同文本（"壁纸轮播已锁定，滑动切换无效" / "可通过滑动切换壁纸"）
  - 开关操作后弹出 Toast 提示
  - 需要 Xposed Hook 生效：Hook `Z1.t.e(MotionEvent)` 阻止背屏壁纸轮播的长按编辑手势

### 区域二：AI 壁纸自定义

- `SuperArrow`：自定义背屏壁纸视频
  - 标题："自定义背屏壁纸视频"
  - 摘要三态：
    - 处理中："处理中…"
    - 未找到壁纸："未找到 AI 壁纸，点击创建"（控件仍可点击）
    - 就绪："点击选择视频"
  - 点击始终打开系统文件选择器（`video/*`），无论是否已有 AI 壁纸
  - 无 AI 壁纸时自动创建（从模板构建 .mrc + 注入 runtime.json）
- `SuperSwitch`：视频循环播放
  - 标题："视频循环播放"
  - 摘要：根据开关状态显示"视频持续循环播放" / "视频播放一次后停止"
  - 切换时修改 MAML manifest.xml 的 loop 属性，写回后重启背屏
  - 未找到壁纸时禁用
- `TextButton`：恢复原始壁纸（始终显示）
  - 点击删除 Janus 自定义壁纸文件，Hook 停止重定向，恢复系统原始壁纸
  - 处理中时禁用
  - 水平内边距 12dp，底部内边距 12dp，等权重

### 交互

- 开启壁纸防打断 -> 保存到 SharedPreferences（应用本地配置文件），Xposed Hook 侧读取后阻止背屏 `SubScreenLauncher.onPause()` 销毁 MAML 壁纸引擎
- 关闭壁纸防打断 -> 保存到 SharedPreferences，恢复正常 onPause 行为
- 开启壁纸锁定 -> 保存到 SharedPreferences，Xposed Hook 侧读取后拦截壁纸轮播长按手势，阻止进入编辑模式切换壁纸
- 关闭壁纸锁定 -> 保存到 SharedPreferences，恢复正常长按编辑行为
- 点击自定义壁纸视频 -> 打开 SAF 视频选择器 -> 先尝试替换已有 AI 壁纸的 .mrc，失败则自动从模板创建新 AI 壁纸条目 -> 写入 Janus 专用路径 `janus_custom.mrc` -> Xposed Hook 重定向 subscreencenter 读取 -> 重启背屏生效
- 切换视频循环 -> 修改 rearscreen ZIP 内 manifest.xml 最后一个 `loop` 属性，写回后重启背屏
- 恢复原始壁纸 -> 删除 `janus_custom.mrc`，Xposed Hook 停止重定向，恢复系统壁纸
- 所有壁纸操作在 IO 线程（后台线程，不会卡住界面）执行，处理中禁用相关控件
